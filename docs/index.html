// quickstart

From install to a gated page

Three steps for you. page-foundry handles everything between the ask and the finished page.

  1. Install the skill.

    npx skills add taylorbanks/page-foundry

  2. Ask for a page.

    /page-foundry build a homepage for my project

  3. Get a page and a gate report.

    It interviews you once, drafts the copy in your voice, runs the voice scan, builds static HTML, and hands back the pass or fail table.

You never touch the internals. During step 3, page-foundry runs a standard-library voice scanner over the draft, and the copy does not ship until it comes back clean. Its rules live in a file you can edit, so your voice and what gets enforced stay in sync.

// how it works

Why the output does not read or look AI-made

A page runs through seven build phases, then eight gates that can each stop it. The voice scan and the design anti-slop list are there to catch what reads or looks AI-made.

Build phases in order

  • -1PreflightDetect companion skills. Install only from pinned sources, only on approval.
  • 0IntakeOne durable brief. It interviews you; a PRD is source material, never the brief.
  • 1Message architecturePositioning, claim hierarchy, objection map, proof inventory.
  • 2Page specOne of eight archetypes fixes section order, CTA policy, proof placement.
  • 3CopyIn your voice, under a scan that rejects the words that mark AI writing.
  • 4Design directionOne token set per property. An anti-slop list rejects the AI-page defaults.
  • 5Build or handoffShip static HTML, or a copy-locked package for a design tool.

What stops the AI look the gates

  • ✓A voice scanrejects the words that mark AI writing, 103 of them, in a file you edit.
  • ✓A pattern passcatches the AI language patterns a word list cannot see, like negative parallelism and forced rules of three.
  • ✓An anti-slop listrejects the visual defaults that give an AI page away, like the purple gradient and the identical-icon-card row.
  • ✓An integrity gaterefuses invented proof, so nothing on the page is fabricated, including the commands it shows.

Every run writes to a per-property log. The next run reads it, so pages learn from the ones before them.

// build or hand off

Build it here, or hand it off

Build mode ships static HTML you own. You do not have to let page-foundry build the page, though.

Handoff mode produces a complete design package instead: the researched brief, the page spec, the final copy marked verbatim so nothing gets paraphrased, the design tokens, and acceptance criteria pulled from the gates. Hand it to Claude Design, Open Design, Codex, Gemini, or whatever you build with, and assemble the page there. The copy has already cleared the voice and pattern gates, so whatever builds it starts from finished work.

Explore mode sits in between: contrasting design directions to pick from before it builds the winner.

// open source

Open, and built on the best

Issues and pull requests on GitHub. The test suite runs before every tag; semantic versions, a changelog per release, currently v2.6.0. Written by a security practitioner who assumes you will not take any of that on faith.

  • marketingskills by Corey Haines: the product-marketing, copywriting, CRO, research, and pricing skills behind the intake and the conversion gate.
  • Anthropic's skills: frontend-design, the reason the output does not look like every AI page; plus theme-factory and skill-creator.
  • web-design-guidelines by Vercel Labs: the accessibility, typography, and UX rules in the design phase and the render gate.
  • gstack by Garry Tan: design consultation, the variant board behind explore mode, and visual review.
  • Remotion, the MECLABS Conversion Sequence, and skills.sh by Vercel, the rails this ships on.
