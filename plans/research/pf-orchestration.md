## Systemic pattern

The skill has exactly **one** real invocation contract (humanizer) and one real consumption loop (foundry-log conversion data). The gate report tracks **absence** (the `degraded` line) meticulously and **quality of presence** not at all: fourteen of seventeen companions can be "invoked" as a token gesture and nothing in any artifact would distinguish that from deep use. The orchestration doctrine says "the companion is the standard" but provides no way to detect when the standard was met. The two highest-differentiation companions (customer-research, marketing-psychology) are the two with no artifacts — precisely the work that separates this skill from a one-shot prompt leaves no trace.

## 5. Ten highest-leverage improvements

1. **Per-companion I/O contract table.** Extend the companion table with four columns: INPUT (exact files/paths handed over), OUTPUT (artifact path + required sections), CONSUMER (the phase that reads it), EVIDENCE (the gate-report line it fills). One table makes every grade above enforceable.
2. **A run artifact directory** — `.agents/foundry/<product>/` with fixed filenames: `voc.md`, `persuasion-map.md`, `message-architecture.md`, `page-spec.md`, `conversion-audit.md`, `tokens.json`, `copy/*.md`. Phases read files, not lingering context; runs become diffable and resumable.
3. **Per-companion evidence lines in the gate report**, modeled on the humanizer line: `cro: invoked, wrote conversion-audit.md, 3 spec changes applied` / `marketing-psychology: skipped (degraded)`. Rubber-stamping becomes visible in the artifact the user actually reads.
4. **VOC provenance schema.** Every quote in `voc.md` carries a source (URL, transcript line, review ID). Gate 2's "every quote traces to the proof inventory" and Gate 8 integrity become mechanical greps against this file instead of assertions. This also closes a live integrity hole: nothing today prevents customer-research output from being paraphrase presented as verbatim.
5. **Independent MECLABS audit.** Run the Gate 1 cro audit in a fresh subagent that receives only the rendered page + brief — not the drafting context or the spec rationale. Record score divergence from the builder's self-score. Today the author grades its own homework.
6. **Parallel multi-candidate generation beyond headlines.** The 6-headline pattern is the only place it exists. Extend: copywriting drafts 2-3 hero-section candidates against different objection framings; frontend-design always produces 2-3 token plans (not just explore mode); each selection logs criteria and reasons to the artifact directory.
7. **Copy-editing changelog + collapse the redundant edit pass.** copy-editing must output a diff/changelog (what was cut, what tightened) that lands in the gate report; assign step 4's "only cuts" pass to copy-editing explicitly or delete it. Also: re-run scan+humanizer after any post-humanizer edit — the current step order (humanize → cut → red-team fixes) lets unscanned prose reach Gate 2.
8. **Move ai-seo and analytics into Phase 2.** The 40-60-word extractable answer block is a copy constraint — it belongs in the spec, not discovered at Gate 6. The conversion event and UTM convention belong in the spec so CTAs are built instrumented. Gates then verify decisions instead of retrofitting them.
9. **Real contracts for the orphans.** competitors feeds Phase 0/1 (the "unlike ALTERNATIVE" clause and competitive frame), not just comparison tables. theme-factory gets an explicit trigger (no brand assets + no DESIGN.md → theme-factory generates the persisted theme) or is cut from the table. pricing's Phase 2 output must also generate Gate 6's `/pricing.md` — currently two disconnected pricing requirements.
10. **Verbatim-copy integrity check in build mode.** Hash or snapshot approved copy blocks at end of Phase 3; after web-artifacts-builder or `/design-html` builds, diff rendered text against the snapshot; any drift fails Gate 2. Handoff mode already has verbatim protection; build mode, the default, has none.

## 6. Too early / too late / never

- **Never:** theme-factory (no invocation point exists in any phase body — table-only companion).
- **Effectively never:** marketing-psychology (invoked with no inputs, no output, no evidence — indistinguishable from not running).
- **Too late — ai-seo:** extractability constraints arrive at Gate 6 after copy is frozen and humanized; retrofitting an answer block post-gate re-triggers the whole voice chain.
- **Too late — analytics:** measurement design at Gate 7 after the page is built; conversion event belongs in the Phase 2 spec.
- **Too late — competitors:** invoked only for comparison sections in Phase 2, but the positioning sentence ("unlike ALTERNATIVE") and competitive frame are locked in Phases 0-1 without it.
- **Too late — customer-research:** runs in Phase 1, after the Phase 0 interview has already asked the user for "the buyer's verbatim language" — the companion could pre-fill and shrink the interview; run it during intake.
- **Mid-pipeline gap — cro:** invoked at spec (Phase 2) and audit (Gate 1) but absent during Phase 3 drafting, where headline candidates are scored against "budget, message match, specificity" without the companion that owns those heuristics.
- **Ordering defect within Phase 3:** humanizer runs at step 3, then step 4 cuts and step 5 red-team fixes edit the copy afterward with no mandated re-scan until Gate 6.
- **Table/body drift:** companion table claims remotion at Phase 2 and gstack at Phase 6 as `/qa`+`/design-review` "additions"; Phase 2 body never invokes remotion, and the gstack additions produce no recorded output.
